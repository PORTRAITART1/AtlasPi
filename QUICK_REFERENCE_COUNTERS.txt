HOW DO THEY WORK?
───────────────────────────────────────────────────────────────────
1. Admin loads pending listings
   → Counters automatically initialize with correct numbers
   
2. Admin moderates a listing (changes status)
   → Page reloads automatically
   → Counters recalculate with new distribution
   
3. Repeat: Counters stay in sync with data

EXAMPLE USAGE FLOW
───────────────────────────────────────────────────────────────────

1. Admin opens page
   → Portal section shows normally
   
2. Admin clicks "Show Admin Moderation"
   → Admin section expands
   → Counters show 0,0,0,0 (waiting for data)
   
3. Admin enters secret key
   → Clicks "Load Pending Listings"
   → Counters update: e.g., 5,3,1,1
   → Admin sees: 5 pending, 3 approved, 1 rejected, 1 suspended
   
4. Admin moderates listing
   → Changes status from "pending" to "approved"
   → Clicks "Apply Moderation"
   → Page reloads automatically
   → Counters update: 4,4,1,1 (one less pending, one more approved)
   
5. Repeat step 4 as needed
   → Counters always in sync

Update Trigger: After listings load + after moderation

COMMON QUESTIONS
───────────────────────────────────────────────────────────────────

Q: Do counters update automatically?
A: Yes! After moderation, page reloads and counters recalculate.

Q: Do I need a new API endpoint?
A: No! Counters use existing data from the DOM.

Q: Can counters be inaccurate?
A: Only if the select values are wrong. They always read from
   the current DOM state.

Q: Do they work on mobile?
A: Yes! They stack into one column on small screens.

Q: What if there are no pending listings?
A: All counters show 0. No errors.

TROUBLESHOOTING
───────────────────────────────────────────────────────────────────

Issue: Counters not showing
Solution: 
  1. Load pending listings first
  2. Wait for listings to appear
  3. Counters should render automatically

Issue: Counter numbers are wrong
Solution:
  1. Verify pending listings loaded correctly
  2. Check browser console for errors
  3. Try reloading the admin section

Issue: Counters disappear after moderation
Solution:
  1. This is normal - page reloads to refresh data
  2. Counters will reappear after reload completes
  3. Usually takes 1-2 seconds
