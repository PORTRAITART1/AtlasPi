QU'EST-CE QUE PiRC2?

PiRC2 = Pi Reference Commerce Protocol 2

C'est le standard officiel Pi Network pour:
  ✅ Authentification Pi (OAuth2 compliant)
  ✅ Paiements Pi (blockchain verified)
  ✅ Validation wallets Pi
  ✅ Consentement utilisateur
  ✅ Sécurité (HTTPS, signatures)

= "Code de conduite commerciale" pour être accepté par Pi Network

IMPACTS CONCRETS

1. AUTHENTICATION
   Avant: uid="test-user-001" (hardcodé, jamais validé)
   Après: uid doit être validé par Pi API (PiRC2 strict)
   → Risque HAUT: tout casse si uid change

2. PAYMENTS
   Avant: paymentId="pi-demo-123456" (fake)
   Après: paymentId signé par Pi, validé blockchain
   → Risque HAUT: demo payments arrêtent de marcher (intentionnel)

3. WALLETS
   Avant: wallet_address = input text libre
   Après: wallet doit être prouvé par utilisateur (PiRC2 ownership proof)
   → Risque MOYEN: merchants perdent accès s'ils ne reprouv pas

4. SÉCURITÉ
   Avant: HTTP localhost possible
   Après: HTTPS obligatoire (PiRC2 requiert)
   → Risque MOYEN: dev local change
